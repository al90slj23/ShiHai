# Conversation Memory Architecture

> **版本**：v0.1.0  
> **最后更新**：2026-05-26

## 核心原则

对话不直接等于长期记忆。ShiHai 将对话拆成两层：

```text
Raw Conversation Source  原始证据层
Canonical Memory         纯化记忆层
```

- 原始层保存“用户/Agent 原来怎么说”。
- 纯化层保存“系统认为这意味着什么”。
- 纯化记忆必须保留 `source_ref`，能回溯到原始对话。

## 两种处理节奏

### 1. 实时轻量捕获

每条消息先追加到 raw log：

```text
selves/<SelfName>/01_Raw/conversations/<source_agent>/<source_ref>.jsonl
```

raw log 是证据，不默认注入 Agent 上下文，不默认当作 canonical memory。

### 2. 周期性纯化整理

会话结束、每 N 分钟、每天或每周运行 consolidation：

```text
raw conversation
  → session summary
  → extracted insights
  → memory proposals
  → review queue / canonical write
```

## 默认写入策略

| 内容类型 | 默认位置 | 审核策略 |
|---|---|---|
| 原始消息 | `01_Raw/conversations/` | 直接追加 |
| 会话摘要 | `02_Processed/conversation_summaries/` | 可自动生成 |
| 事实/偏好/长期目标候选 | `03_Canonical/review_queue/pending_memory.jsonl` | 默认 pending |
| 普通事件 | `03_Canonical/events/` | 可直接写入，但必须带来源 |
| 身份、关系、价值观、健康、财务、第三方推断 | review queue | 必须人工确认 |

## Raw conversation record

```json
{
  "id": "msg_...",
  "source_type": "conversation",
  "source_agent": "hermes",
  "source_ref": "wechat-session-...",
  "speaker": "user",
  "text": "原始对话文本",
  "sha256": "...",
  "created_at": "ISO-8601"
}
```

## Purified memory record

纯化后的 memory 可以是 event / claim / decision / relation / knowledge note。

最低要求：

```json
{
  "id": "claim_...",
  "claim": "提炼后的长期事实或偏好",
  "source_type": "conversation",
  "source_ref": "wechat-session-...",
  "confidence": 0.9,
  "status": "active",
  "created_by": "hermes",
  "created_at": "ISO-8601"
}
```

## 为什么两层都要保留

只保存 raw：噪音太大，难以检索和上下文注入。  
只保存 purified：会丢失语气、上下文和纠错证据。

所以 ShiHai 的原则是：

> 原始对话作为证据完整保留，长期使用的是可审查、可回溯、可更新的纯化记忆。
